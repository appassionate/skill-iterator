# generalization.md Template

The generalization.md captures problems encountered during skill execution, how they were
solved, and the generalized strategies derived from those solutions. This file feeds the
next iteration's todo.md decision-making — it is where specific friction points become
reusable principles.

```markdown
# Generalization: iter.XXXX

## Problem-Solution-Generalization Table

| # | Problem Encountered | How It Was Solved | Generalized Strategy | Recommendation for Next Iter |
|---|--------------------|--------------------|---------------------|-----------------------------|
| 1 | [specific problem] | [specific fix]     | [broader principle applicable beyond this case] | [actionable item or "no action needed"] |

## Strategy Notes

[Free-form discussion of patterns, trade-offs, or design decisions that emerged from
generalization. This is where reasoning about *why* a strategy generalizes belongs.]
```

The "Generalized Strategy" column is the key artifact: it answers "what principle did we
learn that applies beyond this specific case?" The "Recommendation" column becomes a
candidate for the next iteration's todo.md carryover.
